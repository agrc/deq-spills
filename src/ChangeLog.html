DEQ Spills Embedded Map Changelog

2.4.0 Oct 18, 2016

  • Switch to web mercator.

2.2.0 Jul 21, 2014

  • Added INDIAN return attribute (#36).
  • Fixed bug that was causing the "UB" meridian to be selected after clearing the fields (#35).
  • Renamed the zoom button to "Find" (#34).

2.1.1 Jul 1, 2014

  • Added ZIPCITY return parameter (#28).
  • Fixed bug related to passing TRS values into the widget (#30).
  • Decreased delay for find address help popup (#31).
  • Fixed layout issues for IE8 (#26).
  • Fixed server setting for fonts that was causing icons to not show up in IE11.

2.1.0 Jun 12, 2014

  • Salt Lake meridian checked by default (#23).
  • Added city/zip only search help popup (#22).
  • Zip code initialization parameter fixed (#21).
  • Fixed incorrect range direction (#20).
  • Fixed bug causing degrees, minutes, and seconds to not be calculated correctly (#19).
  • Added popup that shows when a user clicks outside of the state of Utah (#18).

2.0.0 Jun 2, 2014

  • Added additional return data including (#4):
    • Zip Code
    • Lat/Long (Decimal Degrees)
    • Lat/Long (Degrees Minutes Seconds)
    • City
    • County
    • deq layer data (DERRID, SITEDESC, ST_KEY)
    • Base Meridian, Township, Section & Range
  • Added additional initial extent definitions including (#5):
    • Lat/Long (Decimal Degrees)
    • Lat/Long (Degrees Minutes Seconds)
    • Base Meridian, Township, Section & Range
  • Added an initialization parameter (layers) to allow the addition of DEQ overlays (#6).
  • Fixed a usability bug with the find address widget (#7).
  • Fixed a layout bug with the map spinner (#9).

1.7.0 Nov 21, 2013

  • Added Section/Township option.

1.6.1 Apr 22, 2013

  • Fixed layout bug with the clear all fields button in IE7.

1.6.0 Apr 3, 2013

  • Added Base Map Selector widget. Redmine 23863
  • "Clear All Fields" button now clears out the map search box. Redmine 23864
  • Disabled alert box error messages. Redmine 23865

1.5.0 Mar 21, 2013

  • Fires onFind event on address match now. Redmine 23584
  • "No Match Found" text changed to red. Redmine 23585
  • Added "Clear All Fields" button. Redmine 23586
  • Fixed bug in IE8 that caused an error when trying to find a route/milepost. Redmine 23589
  • Minor visual improvements.

1.4.0 Mar 19, 2013

  • Added failover to address/route&milepost so that if they fail it tries to match a city/town or county. Redmine 23545
  • Added AGRC Web API key parameter. Redmine 23560

1.3.0 Mar 18, 2013

  • Added map search bar to allow for zooming to cities, towns or counties. Redmine https://redmine.dts.utah.gov/issues/23343
  • Confirmed that parameter hierarchy is XY -> Address/Rt Milepost -> City/Town -> County -> Zoom to State. Redmine

1.2.0 Mar 14, 2013

  • Added cityName parameter. Redmine #23339
  • Added addressStreet and addressZone init parameters. These can be used for both addresses and route/mileposts. See embed-demo.html for an example. Redmine #23341
  • Added find address controls to map widget. Redmine #23342

1.1.0 Mar 13, 2013

  • Added countyName parameter. Redmine #23340
  • Upgraded to latest ESRI JavaScript API (3.3)
  • Lots of general project clean up and polishing.
